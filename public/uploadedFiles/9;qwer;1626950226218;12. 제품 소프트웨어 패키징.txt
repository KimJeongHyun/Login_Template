제품 소프트웨어 패키징
 : 개발이 완료된 제품 소프트웨어를 사용자에게 전달하기 위한 형태로 패키징함

모듈화
 : 모듈을 사용하여 소프트웨어의 품질을 향상시키거나 테스트, 디버깅, 통합, 수정을 용이하게 만드는 설계 기법

모듈화의 장점
: 개발 편의성, 복잡성 감소

릴리즈 노트 
 : 최종 사용자인 고객에서 개발 과정 중 발생한 릴리즈 정보를 전달하도록 기록한 문서

릴리즈 노트 작성항목
 : 헤개목이 재수사소 노면연

제품 소프트웨어 패키징 도구
 : 디지털 콘텐츠의 복제 방지, 불법 사용, 과금 서비스를 통해 정상 사용자를 검증하는 도구를 통해 패키징 해야한다.

패키징 도구 고려사항
 : 암호화/보안, 이기종 연동, 사용자 편의성, 적합한 암호화 알고리즘

저작권 관리 구성요소
 1. 콘텐츠 제공자 : 콘텐츠의 원작자/제작자
 2. 콘텐츠 분배자 : 콘텐츠를 받아 배포하는 자
 3. 패키저 : 콘텐츠를 메타 데이터와 함께 묶어 배포하는 기능
 4. 보안 컨테이너 : 원본을 안전하게 유통하기 위한 보안 장치
 5. DRM 컨트롤러 : 콘텐츠에 대한 사용자들의 접근 권한 관리
 6. 클리어링 하우스 : 소비자와 유통업자 간 디지털 라이센스 중계 및 발급 처리

패키징 도구 세부 기술들...
 : 암 키 식 저 파 정 크 인

암호화
 1. 공개키 기반 구조 : 공개키 기반 암호화를 이용해 네트워크 상에서 안전한 비밀통신을 보장함
 2. 대칭 및 비대칭 암호화 : 암호화키와 복호화키가 같은 대칭키 암호화, 암호화키는 공개되어 있지만 복호화키는 다른 비대칭 암호화
 3. 전자서명 : 서명자를 확인하고 해당 문서에 서명했다는 사실을 알리기 위한 기법

식별 기술
 1. DOI : 디지털 콘텐츠의 보호 및 위치를 추적하기 위한 바코드 시스템
 2. URI : 인터넷 자원을 고유히 식별하도록 하는 기술.

저작권 표현
 1. XrML : XML 기반으로 만들어진 저작권 표현을 다루는 마크업 언어
 2. MPEG-21 : 상호 운용성을 제공하는 멀티미디어 표준 규격

암호화 파일 생성
 1. Prepackaging : 콘텐츠 등록시 바로 암호화 파일 생성
 2. On The Fly packaging : 사용자 요청시 암호화 파일 생성

정책 관리
 1. XML : 특수한 목적을 위해 사용되는 다목적 마크업 언어
 2. CMS : 콘텐츠의 공급 과정을 총괄하는 기술

크랙 방지
 1. 코드 난독화 : 역공학을 통한 코드 해석을 막기 위해 복잡한 구조로 코드 변경
 2. Secure DB : 커널 암호화 방식으로 데이터베이스 암호화

인증
 1. Single Sign On : 한번의 로그인으로 동시에 다른 사이트에서도 로그인할 수 있도록 하는 기술.

제품 소프트웨어 매뉴얼
 : 사용자 측면에서 패키징 이후 설치 및 사용 방법을 알 수 있도록 하는 문서

제품 소프트웨어 매뉴얼 기본 작성 항목
 1. 목차 및 개요
 2. 문서 이력 정보
 3. 설치 매뉴얼 주석
 4. 설치 도구의 구성

제품 소프트웨어 사용자 매뉴얼
 : 소프트웨어 사용 시 사용자에게 필요한 환경 및 제반 사항 등을 기록한 문서

백업의 유형
 1. 전체 백업 : 모든 기존 데이터를 백업
 2. 차등 백업 : 기존에 백업된 데이터 이후 변경된 사항을 백업
 3. 증분 백업 : 정해진 시간 이후에 변경된 사항을 백업
