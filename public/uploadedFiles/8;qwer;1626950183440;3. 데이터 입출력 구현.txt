데이터 모델의 종류
 1. 개념적 데이터 모델 : 일상생활을 추상적, 개념적으로 표현한 모델
 2. 논리적 데이터 모델 : 업무의 모습을 논리적인 표기법으로 형상화한 모델
 3. 물리적 데이터 모델 : 논리적 데이터 모델을 물리적 스키마로 정제한 모델

관계 대수
 : 관계형 데이터베이스에서 원하는 정보, 그리고 정보를 다루는 것을 기술한 절차적 정형 언어
 1. 합집합
 2. 차집합
 3. 교집합
 4. 카디션 프로덕트

순수 관계 연산자
 1. 셀렉트
 2. 프로젝트
 3. 조인
 4. 디비전

논리 데이터 모델의 속성
 1. 개체
 2. 속성
 3. 관계

E-R 모델
 : 현실에 존재하는 데이터와 그 데이터들 간의 관계를 사람들이 잘 이해할 수 있도록 표현한 모델

E-R 다이어그램 기호
 1. 개체 : 동그라미
 2. 조건 : 마름모
 3. 속성 : 동그라미
 4. 다중 값 속성 : 이중타원
 5. 관계 : 선

정규화
 : 데이터의 중복성을 제거하여 이상현상을 방지하고 데이터의 일관성, 무결성을 유지하는 기법.

이상현상 
 1. 삽입 이상
 2. 갱신 이상
 3. 삭제 이상

정규화 종류
 1. 1정규형 : 원자값
 2. 2정규형 : 부분 종속 제거
 3. 3정규형 : 이행 종속 제거 : A->B, B->C, A->C
 4. 보이스 코드 정규형 : 결정자가 후보키가 아닌 함수 종속 제거
 5.. 4정규형 : 다치 종속 제거 : A->B, A->C
 6. 5정규형 : 조인 종속 제거

반정규화
 : 정규화된 데이터에 대해서 통합, 중복, 부분 테이블 추가 등의 작업을 통해 조회 및 연산 등의 성능 향상 및 데이터 테이블의 단순성을 위한 기법

테이블 제약조건 설계
 1. 제한
  : 참조무결성을 저해하는 연산은 제한
 2. 연쇄(Cascade)
  : 특정 데이터가 삭제되었을 때, 그것을 참조하는 데이터 또한 삭제되는 조건

뷰의 속성
 1. REPLACE
 2. FORCE
 3. NOFORCE
 4. WITH CHECK OPTION
 5. WITH READ ONLY

파티션의 종류
 1. 레인지 파티셔닝 : 연속된 숫자를 이용한 파티셔닝
 2. 리스트 파티셔닝 : 특정 파티션에 저장될 데이터가 명시된 경우 사용하는 파티셔닝
 3. 해시 파티셔닝 : 해시 값을 통해 파티셔닝
 4. 컴포짓 파티셔닝 : 레인지, 리스트 파티셔닝 같이 쓰는 방식

파티션의 장점
 1. 성능 향상
 2. 가용성 향상
 3. 백업 가능
 4. 경합 감소

데이터의 종류
 1. 통합 데이터 : 자료의 중복을 배제한 데이터의 모임
 2. 저장 데이터 : 저장 매체에 저장된 데이터
 3. 운영 데이터 : 조직 업무에 필요한 데이터
 4. 공용 데이터 : 공통으로 사용되는 데이터

RDBMS
 : 관계형 모델을 기반으로한 데이터베이스 관리 시스템

HDBMS
 : 데이터를 상하로 계층화하여 관리하는 시스템

NDBMS
 : 데이터를 네트워크 구조로 표현한 시스템

DBMS의 유형 
 1. Key-Value DBMS : 고유한 키에 하나의 값을 가지는 형태의 DBMS
 2. Column Family Data Store DBMS : 컬럼-값 쌍 구조의 데이터가 저장되는 DBMS
 3. Document Store DBMS : 값의 데이터 타입이 문서인 데이터가 저장되는 DBMS
 4. Graph DBMS : 시맨틱/온톨로지 분야에 활용되는, 데이터가 그래프 형태인 DBMS

NoSQL의 유형
 1. Key-Value Store
 2. Column Family Data Store
 3. Document Store
 4. Graph Store

NoSQL의 특성
 1. Basically Available
 2. Soft-state
 3. Eventually consistency

빅데이터
 : 시간내에 처리가능한 양을 넘어서는 대용량의 비정형 데이터 집합

빅데이터의 특성
 : Volume, Variety, Velocity

데이터 마이닝
 : 대규모 데이터를 체계적으로, 자동적으로 분석하여 특정한 패턴이나 특징을 찾는 기법

데이터 마이닝 주요 기법
 1. 분류
 2. 연속
 3. 연관
 4. 데이터 군집화
