시스템 인터페이스 설계서
 : 각 기능의 세부 인터페이스 정보를 정의한 문서

시스템 인터페이스 정의서
 : 각 인터페이스의 세부정보를 구체화한 문서

인터페이스 데이터 표준확인
 : 상호 연계하고자하는 시스템의 인터페이스가 되어야할 데이터 형식 또는 표준을 의미함

컴포넌트 명세서
 : 컴포넌트의 개요, 내부 클래스, 인터페이스를 통해 외부와 통신하는 명세

인터페이스 명세서
 : 인터페이스의 세부 조건, 기능을 명세한 문서

JSON
 : 속성-값 또는 키-값 쌍의 데이터 오브젝트를 사용자가 읽기 쉽도록 텍스트 형식으로 쓴 개방형 표준 포맷

REST
 : 웹과 같은 하이퍼미디어 환경에서 자원의 존재/상태 정보를  HTTP 메서드를 통해 교환하는 웹 아키텍쳐

XML
 : HTML 언어를 개선한 언어로, SGML의 단점을 보완한 특수 목적 마크업 언어

AJAX
 : 자바스크립트를 이용해 웹 서버와 클라이언트 간 비동기적으로 XML 데이터를 교환하고 조작하기 위한 웹 기술

AJAX 주요 기술
 1. XMLHttpRequest : 웹 페이지와 서버 간의 메서드가 데이터를 전송하는 객체 폼의 API. 비동기 통신을 담당하는 자바스크립트 객체
 2. JavaScript : 객체 기반 스크립트 프로그래밍 언어
 3. XML : 개선된 HTML 언어로, SGML 언어를 보완한 특수 목적용 마크업 언어
 4. DOM : XML 구조를 트리 형태로 접근할 수 있도록 하는 API
 5. XSLT : 어떠한 XML 언어를 다른 XML 언어로 변환시켜주는 XML 기반의 언어
 6. HTML : 웹 문서를 표현하는 표준 마크업 언어
 7. CSS : 웹 페이지가 화면 단위로 어떻게 보이는지를 그리는 마크업 언어

시큐어 코딩 가이드
 1. 입력 데이터 검증 및 표현

 5. 코드 오류

DB 암호화 종류
 1. API : 어플리케이션 레벨에서 API를 적용하는 어플리케이션 수정 방식
 2. Plug-In : DB 서버에 암복호화 모듈이 설치된 경우
 3. TDE : DB 커널이 자체적으로 암복호화 기능을 수행함
 4. Hybrid : API, Plug-in 기능이 적절히 섞인 경우

데이터 암호화 전송의 종류
 1. IPSec : 3계층에서 기밀성의 인증헤더, 암호화와 보안의 ESP 알고리즘으로 안전하게 데이터를 통신하는 기법
 2. SSL/TLS : 4~7계층에서 클라이언트와 서버 간 데이터를 암호화하여 기밀성을 보장하고, 데이터의 무결성과 보안을 보장하는 암호화 프로토콜
 3. S-HTTP : 웹 상에서 보내는 모든 메세지를 암호화하여 보내는 방법

인터페이스 구현 검증 도구
 1. xUnit : 다양한 개발 언어를 지원하는 단위테스트 프레임워크
 2. STAF : 다양한 환경을 지원하는 테스트 프레임워크
 3. FitNesse : 웹 기반 테스트 케이스 설계/실행/결과 확인을 지원하는 테스트 프레임워크
 4. NTAF :FitNesse의 협업, STAF의 확장성, 재사용성을 합친 네이버의 테스트 프레임워크
 5. Selenium : 다양한 브라우저와 개발언어를 지원하는 테스트 프레임워크
 6. watir : 루비 기반 테스트 프레임워크

인터페이스 감시 도구
